# QuoteFlow - Sprint 1 开发任务清单

> 目标：打通“客户询盘 → AI 解析 → 生成报价与回复”的核心链路  
> 周期：Sprint 1（建议 5~7 天）  
> 优先级：P0（必须完成）  
> 当前版本：MVP v0.1

---

# 一、Sprint 目标（本周必须做到什么）

本 Sprint 完成后，用户必须可以完成以下流程：

1. 登录系统
2. 进入“新建报价”页面
3. 输入客户询盘文本（可选上传文件）
4. 调用 AI 解析客户需求
5. 展示并编辑结构化参数
6. 生成报价草稿
7. 生成 3 版客户回复话术
8. 保存为草稿
9. 在报价详情页再次打开并继续编辑

> 若以上任一关键步骤无法完成，则本 Sprint 不算完成。

---

# 二、本 Sprint 交付定义（Definition of Done）

## 本 Sprint 只有满足以下条件才算完成：

- 新建报价页完整可用，不依赖 mock 才能跑通主流程
- AI 解析接口可正常返回结构化参数
- AI 生成接口可正常返回报价草稿 + 回复话术 + 参数确认项
- 生成结果可在前端编辑
- 报价可保存到数据库
- 保存后可从详情页重新打开
- 至少通过 5 条真实询盘测试
- 无阻塞性 Bug（无法生成 / 无法保存 / 页面崩溃）

# 五、前端任务

# 5.1 页面：登录页 `/login`

## 目标

用户可以正常登录并进入系统。

## 任务

- 登录表单功能确认
- 登录成功后跳转 `/dashboard`
- token 写入 localStorage
- 登录失败 toast 提示

## 完成标准

## 验收方式

---

# 5.2 页面：新建报价页 `/quotes/new`

## 目标

让用户输入客户询盘并发起生成。

## 任务

- 左侧客户需求输入区可正常使用
- 客户询盘 textarea 支持长文本输入
- 文件上传组件可上传文件（本 Sprint 可先只上传，不解析）
- “解析需求”按钮触发 AI 解析
- “保存草稿”按钮可调用保存接口
- “清空”按钮可重置当前表单
- 未保存离开提醒

## 完成标准

- 输入一段询盘后点击“解析需求”可进入解析状态
- 保存草稿后数据库有记录
- 再次进入详情页可看到保存数据

## 验收方式

- 输入真实询盘测试 3 次

---

# 5.3 中栏：AI 参数提取区

## 目标

展示 AI 解析出的结构化参数，并允许用户修正。

## 任务

- 基础信息卡片展示解析结果
- 产品参数卡片展示解析结果
- 商务参数卡片展示解析结果
- 所有字段可编辑
- 未识别字段高亮显示
- 未确认项汇总提示可显示

## 完成标准

- AI 返回字段后前端能正确映射
- 用户修改字段后状态可保留
- 空值字段显示“未识别”或高亮状态

## 验收方式

- 用 5 条询盘观察字段映射是否正确

---

# 5.4 右栏：生成结果区

## 目标

## 任务

## 完成标准

- 用户编辑报价项后金额正确更新
- 回复内容复制可正常使用

## 验收方式

- 生成 3 份报价，检查展示与编辑逻辑

# 6.2 AI 解析接口 `/api/quotes/parse`

## 目标

将客户原始询盘转换为结构化参数。

## 输入

- 客户原始询盘文本
- 可选语言信息
- 可选上传文件内容（本 Sprint 可为空）

## 输出（建议结构）

```json
{
  "customer_name": "",
  "country": "",
  "currency": "USD",
  "delivery_place": "",
  "product_name": "",
  "model": "",
  "material": "",
  "size": "",
  "color": "",
  "quantity": 0,
  "packaging": "",
  "moq": "",
  "payment_term": "",
  "lead_time": "",
  "validity": "",
  "include_shipping": false,
  "missing_fields": []
}
```

## 任务

- Prompt 输出结构固定
- 接口返回字段统一
- AI 失败时返回可识别错误
- 不能返回不规范 JSON
- 未识别字段留空，不乱编

## 完成标准

- 5 条询盘中至少 4 条能正确返回结构化 JSON
- 不允许接口偶发返回乱码/非 JSON

## 验收方式

- Postman + 前端联调测试 5 条真实询盘

---

# 6.3 AI 生成接口 `/api/quotes/generate`

## 目标

根据结构化参数生成报价草稿和客户回复。

## 输入

- 原始询盘
- 解析结果
- 用户修正后的参数

## 输出（建议结构）

```json
{
  "quote_items": [
    {
      "product_name": "",
      "model": "",
      "spec": "",
      "quantity": 0,
      "unit_price": 0,
      "amount": 0,
      "remark": ""
    }
  ],
  "reply_versions": {
    "short": "",
    "professional": "",
    "followup": ""
  },
  "confirmation_list": [],
  "attachment_list": []
}
```

## 任务

- 生成报价项
- 生成 3 版回复话术
- 生成参数确认清单
- 附件列表可先返回空数组
- AI 错误时返回统一失败信息

## 完成标准

- 返回数据可被前端直接渲染
- 至少生成 1 条报价项
- 回复版本字段完整
- 确认清单为数组结构

## 验收方式

- 5 条询盘生成结果测试

---

# 6.4 报价保存模块 `/api/quotes`

## 目标

把生成结果保存为可复用的报价记录。

## 任务

- POST `/api/quotes` 保存报价
- GET `/api/quotes/:id` 获取报价详情
- PUT `/api/quotes/:id` 更新报价
- Quote / QuoteItem 数据结构检查
- JSON 字段存储规范统一

## 完成标准

- 保存成功后数据库中有完整记录
- 详情页可正确读取并还原
- 更新后数据可持久保存

## 验收方式

- 新建 3 条报价并反复读取更新

---

# 6.5 文件上传接口 `/api/upload`

## 目标

支持用户上传询盘附件。

## 本 Sprint 最低要求

- 文件上传接口可正常上传文件
- 返回文件 URL / 文件名 / 文件路径
- 限制允许类型：pdf/doc/docx/xls/xlsx/png/jpg/jpeg
- 限制单文件大小（建议 10MB）

## 本 Sprint 暂不要求

- 文件内容提取
- OCR
- 图片理解

## 完成标准

- 用户上传后前端能拿到文件信息
- 上传失败时有错误提示

## 验收方式

- 上传 5 种类型文件测试

# 七、数据结构检查（本 Sprint 必须统一）

# 7.1 Quote 表字段检查

- inquiry_text 原始询盘（实现字段：`rawRequirement`）
- parsed_params 解析结果 JSON（实现字段：`parsedParams`）
- reply_versions 回复版本 JSON（实现字段：`replyVersions`）
- confirmation_list 确认项 JSON（实现字段：`confirmationList`）
- attachment_list 附件列表 JSON（实现字段：`attachmentList`）
- status 状态（实现：中文状态如草稿/已发送等）
- total_amount 总金额（实现字段：`totalAmount`）

## 完成标准

- 前后端字段命名一致
- JSON 字段结构固定，不随意变化

---

# 7.2 QuoteItem 字段检查

- product_name（API：`productName`）
- model
- spec（API：`specs`）
- quantity
- unit_price（API：`unitPrice`）
- amount（API：`totalPrice`）
- remark

## 完成标准

- 与前端表格列一一对应
- 可直接编辑回写

---

# 八、AI Prompt 任务（必须做稳定，不求花哨）

# 8.1 解析 Prompt

## 目标

稳定输出结构化 JSON。

## 要求

- 明确要求只返回 JSON
- 明确要求不确定就留空
- 不允许虚构价格、MOQ、交期
- 支持中英混合询盘

## 完成标准

- 返回结构稳定
- 无 markdown 包裹
- 无自然语言解释污染

---

# 8.2 生成 Prompt

## 目标

生成可用报价草稿和客户回复。

## 要求

- 报价项结构化输出
- 回复话术分 3 个版本
- 不能承诺不存在的规格/价格/交期
- 风格偏真实 B2B 销售沟通，不要 AI 味太重

## 完成标准

- 输出内容可直接给前端使用
- 回复话术自然可发
