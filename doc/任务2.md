## 二、P2：数据与回复结构（`需求1.md` 对应）

### 2.1 附件列表拆表

**目标**：将报价上的 `attachmentList`（当前为 `quotes` 表 TEXT JSON，见 `backend/models/quote.go`）逐步迁到独立表（如 `quote_attachments`），便于筛选、审计与权限按行控制。

**任务要点**

- 设计表字段：quote_id、user_id、名称、url、来源（上传/AI 建议/AI 生成）、勾选状态、排序、时间戳等。  
- 双写或一次性迁移 + 读路径切换；前端 `attachmentList` 与后端 DTO 对齐。  
- ZIP / 单附件生成 Job 的入参改为 **稳定 id 列表**，减少依赖整段 JSON 字符串。

**注意事项**

- 与现有 `AttachmentPack.vue`、附件 Job 接口契约一起改，避免「旧报价只有 JSON、新报价只有子表」时逻辑分叉过多；必要时保留 **读时兼容层** 一段周期。

### 2.2 `replyVersions` 等 TEXT JSON 标准化

**目标**：为 `replyVersions`、`parsed_params`、`confirmation_list` 等大字段统一 **版本号**（如 `{ "version": 1, "data": ... }`），并在后端解析失败时有明确降级策略。

**任务要点**

- 定义各结构的 Go struct + 校验函数；写入时带 version。  
- 文档化各 version 差异，便于以后加字段。

**注意事项**

- AI 输出必须先落在「可校验结构」再序列化进库（与任务 1 P0-5 一致）；任务 2 侧重 **历史数据迁移与版本并存**。

## 三、P2：可观测与成本

### 3.1 AI 调用日志与 Token 统计

**目标**：可按用户/按接口/按模型汇总调用次数与估算成本，便于定价与排障。

**任务要点**

- 落库字段建议：user_id、接口名、模型、prompt_tokens、completion_tokens、latency、success/error、quote_id（可选）。  
- 敏感信息脱敏（不要存完整客户原文到日志表，或加密/截断）。

**注意事项**

- 与 `handlers` 里现有 AI 调用点集中封装（避免每个 handler 复制粘贴日志逻辑）。

## 四、P2：运营与组织

### 4.1 管理员后台（最小）

**目标**：用户管理、重置密码协助、查看失败 Job、禁用账号等 **运维必需** 能力。

**任务要点**

- 角色：`admin` vs 普通用户（JWT claim 或独立 admin 表）。  
- 管理接口必须 **独立鉴权**，且禁止与普通用户路由混用同一权限模型而不校验角色。

**注意事项**

- 先 **最小集合**（查用户、改状态、看 Job），避免做成大而全的「第二套产品」。

### 4.2 多租户 / 团队协作（可售文档 §七-1）

**目标**：组织维度（tenant/team）、成员邀请、资源共享边界（产品库、模板是否团队可见）。

**任务要点**

- 数据模型：`tenant_id` 或 `org_id` + 成员关系表；所有资源查询增加组织维度。  
- 与现有 **单用户 `user_id`** 兼容策略（老数据默认「个人组织」）。

**注意事项**

- 这是 **全局横切**；未想清楚权限矩阵前不要只加字段不加固接口，否则易出现跨团队泄露。

## 五、可扩展方向（文档 §七，按产品立项）

以下与可售文档 **§六 商业价值 / §七 可扩展** 对应，适合拆成独立 Epic，**不必一次性做完**。


| 方向        | 说明                  | 与当前栈的关联提示                           |
| --------- | ------------------- | ----------------------------------- |
| CRM 客户管理  | 客户主数据、跟进记录、与报价关联    | 新表 + 报价外键；注意与「仅 quote 内客户信息」字段的合并策略 |
| AI 自动跟进邮件 | 定时/触发式邮件草稿          | 需队列、邮件网关、退订与合规                      |
| 报价数据分析报表  | 转化率、品类、销售漏斗         | 依赖前面 JSON 结构化与拆表，否则统计困难             |
| 多语言自动切换   | UI + 模板 + AI 输出语言策略 | 与模板 `zh/en` 及前端 i18n 统一设计           |


**注意事项**

- §七 每项都可单独 **报价/收费**；开发上优先接 **CRM + 报表** 的数据底座（客户实体 + 干净字段），再叠智能化。
